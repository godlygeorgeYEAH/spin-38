# 4.1 DEPRECATE - Apuesta Mínima Automática

**Fecha de implementación:** 5 de Diciembre de 2025
**Prioridad:** Media
**Estado:** ✅ Completado
**Categoría:** Deprecación / Limpieza de Código

---

## Descripción

Deprecación y remoción del sistema de apuesta mínima automática que asignaba automáticamente el valor más pequeño de ficha (`coinValues[0]`) a apuestas en $0. Este comportamiento restringía el control del usuario sobre sus apuestas.

---

## Razón de la Deprecación

El sistema anterior asignaba automáticamente una apuesta mínima en dos escenarios:

1. **Al cambiar de animal seleccionado**: Si el animal anterior tenía apuesta $0, se le asignaba automáticamente `coinValues[0]`
2. **Antes de girar la ruleta**: Si el animal actual tenía apuesta $0, se le asignaba automáticamente `coinValues[0]`

### Problemas identificados:
- ❌ **Falta de control del usuario**: Apuestas se agregaban sin consentimiento explícito
- ❌ **Comportamiento confuso**: Usuario no esperaba que se agregaran fichas automáticamente
- ❌ **Inconsistente con UX moderna**: Los usuarios deben tener control total sobre sus acciones
- ❌ **Innecesario**: El sistema ya valida que la apuesta total sea > 0 antes de girar

## Código Removido

### 1. En `setCurrentEditingAnimal()` (home.page.ts:179-186)

**ANTES:**
```typescript
public setCurrentEditingAnimal(animal: Animal): void {
  let newSelectedAnimals = this.selectedAnimals;
  if (this.currentEditingAnimal && this.currentEditingAnimal.name !== animal.name) {
    const index = newSelectedAnimals.findIndex(bet => bet.animal.name === this.currentEditingAnimal!.name);
    if (index >= 0 && newSelectedAnimals[index].amount === 0) {
      newSelectedAnimals = newSelectedAnimals.map((bet, i) =>
        i === index ? { ...bet, amount: this.coinValues[0] } : bet
      );
    }
  }
  this.selectedAnimals = newSelectedAnimals;
  this.currentEditingAnimal = animal;
  this.isBettingControlsVisible = true;
  this.updateTotalBetAmount();
}
```

## Impacto en la Funcionalidad

### Comportamiento Anterior:
1. Usuario selecciona Animal A → Se crea apuesta en $0
2. Usuario selecciona Animal B → Animal A automáticamente recibe `coinValues[0]` (ej: $1)
3. Usuario presiona girar → Si hay animal con $0, recibe `coinValues[0]`

### Comportamiento Nuevo:
1. Usuario selecciona Animal A → Se crea apuesta en $0
2. Usuario selecciona Animal B → Animal A permanece en $0
3. Usuario debe agregar fichas manualmente a cada animal
4. Si intenta girar con apuesta total = $0 → Bloqueado por validación existente

### Validaciones que Permanecen:
- ✅ `canSpin()`: Verifica que haya al menos un animal seleccionado
- ✅ `totalBetAmountSubject.value === 0`: Previene giro sin apuestas
- ✅ Sistema de balance: Verifica fondos suficientes
