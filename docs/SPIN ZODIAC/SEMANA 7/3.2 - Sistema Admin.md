# 3.2 Sistema Admin

**Prioridad:** Alta
**Estado:** ✅ Completado
**Categoría:** Nueva Funcionalidad / Administración
**Fecha de Implementación:** 5 de Diciembre de 2025

---

## Descripción

Sistema de administración basado en comandos de consola que permite autenticación segura y acceso a datos privilegiados del juego. Este sistema reemplaza al sistema legacy `setDevPassword()`.

---

## Características Implementadas

### 1. Autenticación Segura
- **Almacenamiento:** SessionStorage (se borra al cerrar navegador)
- **Duración de sesión:** 8 horas
- **Seguridad:** SHA256 + salt
- **Credenciales por defecto:**
  - Usuario: `admin`
  - Contraseña: `ruleta2025`

### 2. Comandos Disponibles

| Comando | Descripción | Ejemplo |
|---------|-------------|---------|
| `adminLogin(username, password)` | Iniciar sesión | `adminLogin("admin", "ruleta2025")` |
| `adminStatus()` | Ver estado de sesión actual | `adminStatus()` |
| `adminChangePassword(currentPass, newPass)` | Cambiar contraseña | `adminChangePassword("ruleta2025", "nuevaPass123")` |
| `adminResetPassword()` | Resetear a contraseña por defecto | `adminResetPassword()` |
| `adminLogout()` | Cerrar sesión | `adminLogout()` |

### 3. Control de Acceso a Logs
Los logs del juego solo se muestran cuando hay un administrador autenticado:
- `Game data before spin` - Solo visible para admins
- `Game data after spin` - Solo visible para admins

## Seguridad

### Implementación de Seguridad

1. **Hash de contraseña:**
   - Algoritmo: SHA256
   - Salt único por cada cambio de contraseña
   - No se almacenan contraseñas en texto plano

2. **Almacenamiento:**
   - SessionStorage (se borra al cerrar navegador)
   - No persiste entre sesiones del navegador

3. **Expiración:**
   - Sesiones expiran automáticamente después de 8 horas
   - Verificación de expiración en cada operación

4. **Validaciones:**
   - Nueva contraseña debe tener mínimo 8 caracteres
   - Verificación de contraseña actual antes de cambiarla
   - Autenticación requerida para todas las operaciones sensibles

### Datos de Sesión Almacenados

```typescript
{
  username: string;           // "admin"
  passwordHash: string;       // SHA256(password + salt)
  salt: string;              // Salt aleatorio
  loginTime: string;         // ISO timestamp
  expiresAt: string;         // ISO timestamp (loginTime + 8h)
  isAuthenticated: boolean;  // true
}
```

## Migración desde Sistema Legacy

### Sistema Antiguo (DEPRECADO)
```javascript
setDevPassword('contraseña')
// Almacenaba en localStorage
// Sin expiración de sesión
// Sin comandos de gestión
```

### Sistema Nuevo (ACTUAL)
```javascript
adminLogin("admin", "ruleta2025")
adminStatus()
adminChangePassword("actual", "nueva")
adminResetPassword()
adminLogout()
// Almacena en sessionStorage
// Sesión de 8 horas
// Comandos completos de gestión
```

**Nota:** El sistema legacy `setDevPassword()` aún funciona pero se marcará como deprecado y se eliminará en una versión futura (ver tarea 4.2).
