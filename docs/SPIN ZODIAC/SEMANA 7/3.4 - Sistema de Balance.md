## Descripción

Sistema robusto que maneja automáticamente las actualizaciones de balance durante el juego. El sistema gestiona apuestas y ganancias de forma automática y confiable, validando fondos suficientes antes de permitir apuestas y manteniendo un historial completo de transacciones.

Interface que define los tipos de transacción y la estructura de datos para el historial.

export enum TransactionType {
  BET = 'bet',       // Apuesta realizada
  WIN = 'win',       // Ganancia obtenida

export interface Transaction {
  id: number;
  type: TransactionType;
  amount: number;
  balanceBefore: number;
  balanceAfter: number;
  timestamp: Date;
  description?: string;
}

#### Métodos Privados Agregados:

**`deductBet(amount: number): void`**
- Deduce una apuesta del balance del jugador
- Registra transacción tipo BET con monto negativo
- Actualiza el BehaviorSubject para reflejar cambios en UI

**`addWinnings(amount: number): void`**
- Agrega ganancias al balance del jugador
- Registra transacción tipo WIN
- Actualiza el BehaviorSubject

**`recordLoss(betAmount: number): void`**
- Registra una transacción de pérdida
- No modifica el balance (ya fue deducido con la apuesta)

**`setBalance(amount: number): string`**
- Establece el balance a un valor específico (comando admin)
- Validaciones: número válido, positivo, entero

**`addBalance(amount: number): string`**
- Agrega/quita una cantidad al balance (comando admin)
- Validaciones: número válido, entero, no resulta en balance negativo

## Comandos Admin

El sistema incluye comandos admin para gestionar el balance del jugador:

### Comandos Disponibles:

#### 1. **`adminGetBalance()`**
Ver el balance actual del jugador.
```javascript
adminGetBalance()
// Output: 💰 Balance actual del jugador: $10000
```

#### 2. **`adminSetBalance(monto)`**
Establecer el balance a un valor específico.
```javascript
adminSetBalance(50000)
// Output: ✅ Balance actualizado: $10000 → $50000
```

**Validaciones:**
- Debe ser un número válido
- No puede ser negativo
- Debe ser un número entero

#### 3. **`adminAddBalance(monto)`**
Agregar o quitar una cantidad del balance.
```javascript
adminAddBalance(5000)  // Agregar $5000
// Output: ✅ Balance actualizado: $10000 +5000 = $15000

adminAddBalance(-2000) // Quitar $2000
// Output: ✅ Balance actualizado: $15000 -2000 = $13000
```

**Validaciones:**
- Debe ser un número válido
- Debe ser un número entero
- No puede resultar en balance negativo

#### 4. **`adminResetBalance()`**
Resetear el balance al valor por defecto ($10000).
```javascript
adminResetBalance()
// Output: ✅ Balance reseteado a: $10000
```

#### 5. **`adminGetTransactions(limit?)`**
Ver el historial de transacciones. Opcionalmente puede especificar un límite para ver solo las últimas N transacciones.
```javascript
adminGetTransactions()      // Ver todas las transacciones
adminGetTransactions(10)    // Ver últimas 10 transacciones
// Output: Tabla formateada con historial
```

**Formato de salida:**
| ID | Tipo | Monto | Balance Antes | Balance Después | Fecha | Descripción |
|----|------|-------|---------------|-----------------|-------|-------------|

#### 6. **`adminClearTransactions()`**
Limpiar todo el historial de transacciones.
```javascript
adminClearTransactions()
// Output: ✅ Historial de transacciones limpiado
```

## Flujo de Transacciones

### Escenario 1: Apuesta Sin Ganancia
1. Usuario selecciona animales y agrega fichas
2. Usuario presiona girar
3. **Validación:** Sistema verifica fondos suficientes
4. **Deducción:** Balance disminuye por el monto apostado
5. **Transacción BET:** Se registra con monto negativo
6. Rueda gira y determina resultado
7. Si no hay ganancia: **Transacción LOSS** se registra
8. UI se actualiza automáticamente

### Escenario 2: Apuesta Con Ganancia
1. Usuario selecciona animales y agrega fichas
2. Usuario presiona girar
3. **Validación:** Sistema verifica fondos suficientes
4. **Deducción:** Balance disminuye por el monto apostado
5. **Transacción BET:** Se registra con monto negativo
6. Rueda gira y determina resultado
7. Si hay ganancia: **Transacción WIN** se registra
8. **Incremento:** Balance aumenta por el monto ganado
9. UI se actualiza automáticamente

### Escenario 3: Fondos Insuficientes
1. Usuario intenta apostar más de su balance
2. **Validación:** Sistema detecta fondos insuficientes
3. **Bloqueo:** Apuesta no se procesa
4. **Error:** Mensaje de error se muestra
5. Usuario debe reducir apuesta o agregar balance (admin)

## Criterios de Aceptación

- ✅ **Balance se actualiza automáticamente después de cada apuesta**
  - Implementado en `spinWheels()` con `deductBet()`

- ✅ **Balance se actualiza correctamente al ganar**
  - Implementado con `addWinnings()` después de calcular resultado

- ✅ **No permite apuestas sin fondos suficientes**
  - Validación implementada antes de procesar apuesta
  - Mensaje de error claro para el usuario

- ✅ **No hay inconsistencias en el balance**
  - Todas las transacciones registradas con balance antes/después
  - BehaviorSubject asegura sincronización

- ✅ **UI refleja el balance actual en todo momento**
  - Display en header con `balance$ | async`
  - CSS estilizado con colores dorados
  - Actualización automática con `ChangeDetectorRef`

- ✅ **Funciona correctamente en casos edge**
  - Balance = 0: No puede apostar
  - Balance < apuesta: Bloqueado con mensaje
  - Ganancia muy grande: Sin problemas numéricos

## Consideraciones Técnicas

### Reactivity
- Uso de `BehaviorSubject` para reactividad
- Pipe `async` en template para suscripción automática
- `ChangeDetectorRef.detectChanges()` para forzar actualización

### Validaciones
- Todas las operaciones de balance validadas
- Tipos de datos verificados (number, integer)
- Límites verificados (no negativo, no insuficiente)

### Persistencia
- **NO hay persistencia** en esta implementación
- Balance se resetea al recargar página
- Transacciones se pierden al cerrar pestaña
- Futuro: Implementar con backend o localStorage

### Performance
- Observable permite updates eficientes
- Solo componentes suscritos se actualizan
- Transacciones almacenadas en memoria
